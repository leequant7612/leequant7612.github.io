Posted on 2022-05-17 In Web Programming , Java

자료실

파라미터 받는 로직은 중복이 많아서 생략함

  • java

    • controller.dataroom

      • DownloadController : 포워드 X

        • request 경로1 : List.jsp => DownloadController
        • request 경로2 : View.jsp => DownloadController

    • EditController : /DataRoom14/Message.jsp 로 포워드

      • request 경로 : View.jsp => PasswordController => EditController

      • (이어서) : Edit.jsp => EditController

      • IndexController : /DataRoom14/index.jsp 로 포워드

    • ListController : /DataRoom14/List.jsp 로 포워드

      • reqeust 경로

    • PasswordController : 수정일 땐 EditController로 포워드 & 삭제일 땐 /DataRoom14/Message.jsp로 포워드

      • View.jsp => PasswordController (모드 = 수정 or 삭제)

      • ViewController : /DataRoom14/View.jsp 로 포워드

        • request 경로1 : Message.jsp (입력,삭제) => ViewController
        • request 경로2 : List.jsp => ViewController

      • WriteContoller : /DataRoom14/Message.jsp 로 포워드

        • request 경로1(GET) : List.jsp => WriteContoller
        • request 경로2(POST) : Write.jsp => WriteContoller

    • model

      • PagingUtil

        • pagingBootStrapStyle(pagingMap) : 아래 그림에 해당하는 HTML 문자열 반환

        • setMapForPaging(map, dao, request) : map에 paing 정보 셋팅

    • model.dataroom

      • DataRoomDAO : JDBC와 연동해서 하는 작업들을 모은 클래스
      • DataRoomDTO : 데이터를 받는지 자바빈 클래스
      • FileUtils : 업로드/다운로드/삭제 메소드를 제공하는 클래스

  • webapp

    • DataRoom14

      • Edit.jsp :

        • 경로 : View.jsp => PasswordController => EditController => Edit.jsp

      • List.jsp

        • 경로 : Top.jsp => ListController => List.jsp

      • Message.jsp

        • 경로1 : Write.jsp => WriteContoller => Message.jsp
        • 경로2 : Edit.jsp => PasswordController => EditController => Message.jsp
        • 경로3 : Edit.jsp => PasswordController => Controller => Message.jsp

      • View.jsp

      • Write.jsp
