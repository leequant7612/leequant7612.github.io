1. 웹의 개요

  • 웹 서버는 HTTP프로토콜로 TCP/IP 네트워크 상에서 HTML을 제공한다.
  • 웹 브라우저는 웹서버에서 제공한 HTML 태그를 분석해 화면을 구성한다.

더 자세히 웹 동작 구조를 설명하면

  1. 웹브라우저에서 https://도메인/웹문서 로 입력한다
  2. DNS 서버는 도메인을 받고 웹브라우저에게 IP를 제공한다.
  3. 웹브라우저는 80번 포트(웹서비스 포트)로 request를 보낸다.
  4. 웹서버는 request를 분석해서 웹문서를 반환하고 클라이언트에게 전송한다.
  5. 브라우저는 웹문서를 해석하고 화면을 띄운다.

2. 네트워크

  • 컴퓨터 네트워크
    • 프로토콜 : 컴퓨터 간의 연결 규격
      • OSI/ 7계층
      • TCP/IP : OSI 7계층을 단순화하여 4계층으로 정의한 프로토콜
        • TCP : 데이터 흐름 관리, 데이터 정확성 확인
        • IP : 데이터패킷을 목적지까지 전송하는 역할
  • IP 주소는 TCP/IP로 연결된 네트워크 상에서 컴퓨터를 구분하는 역할을 한다.
    • 수많은 컴퓨터에 고유한 IP를 할당할 수 없다.
    • 그래서 일반적인 컴퓨터나 스마트폰은 DHCP로 사설IP를 사용한다.
    • Network Access Translator는 사설IP를 공인IP로 변환해준다.
  • 인터넷 : 전세계가 하나로 연결된 컴퓨터 네트워크
    • TCP/IP 프로토콜로 연결된 인터넷 기반의 서비스들
      • 웹
        • HTTP/HTTPS 프로토콜
        • 80/443 포트
      • 이메일 : 이메일 서비스
        • 웹 기반으로 대체되는 추세
        • 25/110/143 포트
      • FTP : 파일 전송 서비스
        • FTP 프로토콜
        • 21 포트
      • TELNET/SSH : 원격 로그인 서비스
        • TELNET/SSH 프로토콜
        • 22/23 포트
      • DNS : 도메인 이름 변환 서비스
        • DNS 프로토콜
        • 53 포트
  • 포트는 하나의 컴퓨터에서 여러개의 네트워크 서비스를 제공할 때 구분하기 위한 목적
    • 포트에도 전용 프로토콜이 존재하여 일치하지 않는 경우 서비스 제공하지 않음

3. 웹 프로그래밍의 구조

  • 클라이언트 : 웹 서비스를 받는 사용자
  • 서버 : 웹 서비스를 제공하는 공급자
  • 프론트엔드 : 클라이언트 중심의 프로그래밍 영역 (HTML, CSS, JS)
    • HTML : 웹 문서의 구조 및 컨텐츠를 표현하는 언어
    • CSS : HTML의 스타일을 정의하는 규격
    • JS : 브라우저가 해석하는 프로그래밍 언어
  • 백엔드 : 서버쪽 프로그래밍 영역 (서블릿, JSP, 스프링, 파이썬)
  • 데이터베이스

3.1 웹 개발 종류

3.1.1 백엔드 중심 개발

  1. 클라이언트가 URL 형태로 요청을 보낸다.
  2. 서블릿은 데이터베이스에 연결하고 JSP로 페이지 이동을 한다.
  3. JSP를 통해 HTML 문서를 출력한다.
  4. 클라이언트에게 HTML을 응답으로 보낸다.

3.1.2 프런트엔드 중심 개발

  1. 클라이언트가 URL 형태로 요청을 보낸다.
  2. 서블릿/JAX-RS/Node.js 는 데이터베이스에 연결하고 JSON을 출력한다.
  3. 클라이언트는 JSON 데이터를 화면에 렌더링한다.

필요한 부분의 데이터만 갱신되므로 실시간 데이터 갱신이 자유롭다.

3.2 트렌드

  • 자바에서 코틀린으로 넘어가는 추세이다.
  • REST API 형태의 개발이 중심을 이루면서 백엔드에서 화면을 다루는 경우가 줄어든다.
  • 서버에서 JSON 데이터만 받고 클라이언트에서 렌더링하는 방식으로 넘어가고 있다.
  • 스프링은 상용화할 수 있는 수준의 대규모 서비스를 개발하는 자바 기반의 백엔드 프레임워크
