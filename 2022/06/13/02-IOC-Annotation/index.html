02_IOC+Annotation

Posted on 2022-06-13 In Web Programming , Java Spring

1. Inversion Of Controll

순제어는 필요한 곳에서 직접 생성하여 사용하는 것을 의미한다.

IOC : 컨테이너가 객체 생성부터 소멸 까지의 라이프사이클을 관리하게 되어 “제어권이 넘어갔다를 의미한다”.

Spring은 필요로하는 왠만한 객체를 컨테이너가 관리한다.

DI : 의존성 주입은 IOC 컨테이너에 있는 객체(의존성)를 주입하는 행위이다.

컨트롤러에서 사용하는 대부분의 객체는 DI로 얻어진다.

AOP : DI보다 더 일반적인 프로그래밍 패러다임으로 핵심로직(메소드) 전후에 Advice(공통기능)를 위빙(주입)하는 방식이다.

1.1 @Autowired로 Bean Wiring (권장)

와이어링은 컨트롤러를 생성할 때 DI를 통해 기존 객체를 연결짓는 것을 의미한다.

@Autowired를 활용하면 빈 설정파일에서 생성자 호출이 필요없다.

@Autowired은 서블릿 컨테이너 설정파일에서 생성한 빈들 중에서

  1. 타입기반으로 인자를 찾는다.
  2. 타입에 해당하는 객체가 2개 이상인 경우 빈의 id를 기반으로 찾는다.
    • id명 <==> 컨트롤러 멤버변수명
  3. id로도 구분이 안된다면 qualifier 태그 활용한다.
  4. 그것도 없으면 500에러가 발생한다. 주입이 꼭 필요하지 않다면 required=false를 인자로 넣는다.

옵션1 @Autowired; @Qualifier(선택사항)

옵션2 (@Resource)

  • 좀 더 엄격한 방식이다.
  • 멤버변수명과 빈설정파일에서 id 속성이 일치할 때만 사용 가능

DI로 받은 빈을 세터로 값 주입 시 원본이 바뀌니 이러한 프로그래밍은 지양하자

1.2 세터를 활용한 주입 (비추천)

  • @Resoruce(name="아이디")세터를 호출하면서 인자로 “아이디”에 해당하는 빈을 인자로 넣는다.
  • 빈 객체에 추가로직을 더해서 설정하고 싶을 때 세터로 주입을 하는 편이다.

2. Annotation @

IOC만 설명하고 넘어가기엔 컨텐츠가 부족하여 다른 Annotation들을 설명하겠다.

2.1 @RequestMapping

클래스에 상위 경로를 붙이는 경우 메소드에선 하위 경로만 적어주면 된다.

2.1.1 여러요청을 하나의 컨트롤러 메소드로 처리

  1. value={"요청URL1", "요청URL2", ...}, method={RequestMethod.GET, RequestMethod.POST}

  2. @PathVariable 활용

    REST API 서버에서 자주 사용하게 될 예정

2.2 @RequestParam

서블릿 API 사용을 줄이고 @RequestParam 사용을 권장한다.

  • 코드 간결
  • 형 변환 필요 없음

함수의 인자명과 파라미터명이 다른 경우

  • @RequestParam("req파라미터명") int arg1
    • req파라미터명이 req에 없다면 bad request(404) 에러가 발생한다.
  • @RequestParam("req파라미터명" required=false) int arg1
    • 404 에러는 발생하지 않는다.
    • default로 값을 넣어주고 싶다면 defaultValue=?를 annotation의 인자로 추가하면 된다.

파라미터 하나의 키에 값이 여러개인 경우

  • @RequestParam String[] inter
    • 단순하게 자료형만 입력하면 맨앞의 값만 받기 때문에 배열로 받아야 한다.

2.2.1 @RequestParam Map map

  • req파라미터가 굉장히 많을 때 사용한다.

  1. DTO 만들기 (속성명 = req파라미터명)
  1. 함수
  1. 뷰페이지

미리 컨테이너에 빈으로 등록하고 함수에 DI를 적용하면 유용할 수도 있다.

2.3 @Resource

  • 기존 빈객체를 컨트롤러에 넣는게 아닌 경우에 세터로 DI를 하게 된다.

2.4 @Lazy

첫 요청을 받았을 때 컨트롤러가 생성된다.

빈 설정파일에 넣어서 생성할 때 지연을 주고 싶다면 <beans:bean lazy-init='true'>로 줄 수도 있다.

2.5 스프링에 빈 등록

스프링에 빈으로 등록하여 다른 곳에 DI를 할 때

  • Configuration 또는
  • Component

를 사용하면 된다.

이를 활용하면 빈 설정파일에 직접 등록하지 않아도 되니깐 빈 관리하기 쉽다

  • @Component

    • 개발자가 직접 작성한 클래스 한개를 등록한다.

  • @Configuration + @Bean

    • @Service와 동일하다
    • 관련있는 빈들끼리만 모으면 관리가 용이하다,

2.6 영역에 데이터 저장

  • @SessionAttribute : 세션 영역에 attribute를 저장할 때 사용한다.
