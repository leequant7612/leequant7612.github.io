1. Spring MVC

  • 스프링 프레임워크에 내장된 웹 프레임워크
  • MVC 패턴에 기반한 어플리케이션 생성
  1. 프론트컨트롤러가 요청을 핸들러매핑에게 전달
    • URL을 컨트롤러 ID로 반환
  2. 프론트컨트롤러가 요청을 컨트롤러에게 전달
    • 모델을 사용해 값을 구하고 Scope Object에 넣고 뷰정보 반환
  3. 프론트컨트롤러가 요청을 뷰리졸버에게 전달
    • 뷰 정보를 물리적 주소로 반환
  4. 프론트컨트롤러가 뷰생성하고 사용자에게 전달
    • JSP 템플릿 엔진 활용

1.1 프론트 컨트롤러 (디스패처 서블릿)

  • 설정파일 위치
    • /WEB-INF/spring/appServlet/servlet-context.xml
  • 디스패처는 하나의 context 컨테이너로 생각할 수 있다.
  • 일반 컨트롤러와 가장 큰 차이는 url을 패턴으로 지정하느냐 값으로 지정하느냐이다.
    • url = *.do (프론트컨트롤러)
    • url = abc.do (일반컨트롤러)

1.2 컨트롤러

  • @Controller로 컨트롤러 클래스를 만들고

    • @RequestMapping("URL")로 내부에 컨트롤러 메소드 만들기

      • 인자

        @RequestParam Map, @RequestParam String, @RequestParam int, …

        ​ 요청파라미터 매핑

        Map, Model, ModelMap

        ​ request 영역에 저장할 때 활용

      • 반환

        String, ModelAndView(옛날방식)

Annotation Handler Mapping (권장, 여러메소드 가능)

SimpleUrlHandlerMapping (옛날방식)

1.3 뷰 리졸버

WEB-INF/spring/appServlet/servlet-context.xml

  • 컨트롤러 클래스 내부의 컨트롤러 메소드는 뷰리졸버의 prefix와 suffix를 고려하여 문자열을 반환한다.
    • /WEB-INF/views/
    • .jsp
  • 뷰리졸버의 영향을 받지 않으려면 컨트롤러 메소드의 반환 문자열 앞에 forward: 또는 redirect:를 붙이자.
    • “forward:ArbitraryController.do”
    • “forward:ArbitraryView.jsp?myParam=123”
    • 참고로, redirect의 url을 .jsp로 두면 클라이언트에서 .jsp를 직접 요청하는 구조라 안됨
