HTML 템플릿

파일 구조

  • src/
    • core/ : 핵심 기능을 포함하거나 추상클래스들을 담은 다른 패키지에서 사용될 클래스들
      • api.ts
      • router.ts
      • view.ts
    • page/ : 뷰페이지 단위로 모듈화 된 패키지
      • news-detail-view.ts
      • news-feed-view.ts
    • types/
      • 프로젝트에서 사용할 타입들을 모은 패키지
    • app.ts : 실행할 메인 파일
    • config.ts : 상수 관리

기존 app.ts는

  • 타입 선언부

    • View
    • Router
    • NewsFeedView
    • NewsDetailView

  • 실행부

STEP 1 : 메인 영역만 살리기

우선 app.tst 전역변수와 실행부만 살리자.

  • CONTENT_URL, NEWS_URL은 상수와 같이 정적으로 쓰일 예정이므로 config.ts로 빼자.

  • store은 다른 모듈에 있는 기능에서 동적으로 업데이트 될 예정이므로 전역변수로 선언하자.

관행일 뿐 정답은 정해져있는 게 아님…

STEP 2 : 타입들 분리하기

types/index.ts에 타입선언부 내용을 따로 빼자

STEP 3 : core

core/ : 핵심 기능을 포함하거나 추상클래스들을 담은 다른 패키지에서 사용될 클래스들

  • api.ts : ajax로 서버에 요청을 보내고 데이터를 받는다.
  • router.ts : 생성과 동시에 hashchange의 이벤트 리스너로 본인의 메소드 route를 등록한다.
  • view.ts

STEP 4 : page

page/ : 뷰페이지 단위로 모듈화 된 패키지

  • index.ts

  • news-detail-view.ts

  • news-feed-view.ts

각자의 뷰페이지에서 사용할 템플릿을 맨위에 정의하고 생성할 때 활용하게 된다.
