3.1 Mapper

[1] resultMap (옵션)

resultMap : select 태그를 사용하여 얻은 조회결과를 담을 자료구조

  • type : DTO 클래스명 (혹은 alias)
  • id : 앞으로 resultMap을 부를 명칭

resultMap을 활용하면, 글과 댓글을 inner join 안해도 댓글을 얻을 수 있다.

[2] Query

<select>

  • id : SQL 쿼리 identifier
  • parameterType : 거의 Map을 쓰는 편이다.
  • resultType : 결과값들의 데이터 타입
    • Primitive 혹은 Primitive DTO
    • DTO의 속성명과 테이블의 컬럼명이 대소문자 수준의 차이라면 문제 없음
  • resultMap : 결과값들의 데이터 타입
    • 일반적인 DTO

<insert>, <delete>, <update>

  • 변경 SQL 문을 하고 난 뒤에 결과값을 저장하고 싶다면 (ex : 추가된 글의 번호)
  • <selectkey order="BEFORE">를 내부에서 활용하면 된다.

SQL 쿼리 작성은 /src/main/resources/onememo/mybatis/mapper에 있는 xml 파일들 보고 연습하길 바람
