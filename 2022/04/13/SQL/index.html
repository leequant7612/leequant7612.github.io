SQL : RDBMS의 데이터를 제어하기 위한 표준 질의어

NUMBER

  • BINARY_INTEGER : (4Byte:-2^31-1 ~ 2^31-1)
  • NUMBER : 38자리의 정수,실수
  • NUMBER(n) : n자리의 정수
  • NUMBER(p,s) : p자리의 실수, 소수점은 s자리 (소수점이 길어지면 반올림 처리)

CHAR

  • CHAR(n) : n(<2000) 바이트 “고정” 문자열
  • NCHAR(n) : n(<1000) 길이의 문자열
  • VARCHAR2(n) : 가변 길이의 문자열, 최대 n(<4000) 바이트
  • NVARCHAR2(n) : 가변 길이의 문자열, 최대 n(<2000) 길이
  • LONG : 가변 길이의 문자열, 최대 4GB
  • …

FROM -> WHERE -> GROUP BY -> HAVING -> SELECT -> ORDER BY

SELECT 컬럼이름,컬럼이름,.....

                    [ORDER BY 데이터 정렬 표현]

    • = ,> ,>=, < , <=, !=
  • 논리연산자
    • AND, OR, NOT
  • SQL연산자
    • BETWEEN a AND b
    • IN, NOT IN
    • LIKE
      • % : 임의의 0개 이상의 문자열
      • _ : 임의의 한 글자
      • LIKE ‘A%’ : A, ABC, A123, …
      • LIKE ‘%A’ : A, BCA,123A, …
      • LIKE ‘%KIM%’ : KIM, ABCKIM ,ABCKIMDEF,KIMABC, …
      • LIKE ‘_A%’ : DA, AA, BACKCL, …
    • IS NULL, IS NOT NULL

2 DDL - 데이터 구조 정의

Create(생성), Alter(수정), Drop(삭제)

  • 테이블 뿐만 아니라
  • 유저, 롤, 인덱스, 뷰, 시퀀스 등을 (재)정의 할 때 사용하는 명령어이다.
  • 추가로, 제약조건 삭제 시 Drop 쓴다.

2.1 CREATE - DDL

테이블(및 다른 객체들)을 생성한다.

2.1.1 Constraints

거의 테이블 생성 시에만 쓴다!!

  • NOT NULL
    • [constraint 제약조건명] not null
  • UNIQUE
    • [constraint 제약조건명] unique
  • PRIMARY KEY : NOT NULL + UNIQUE + 테이블에서 딱 한 칼럼만 가능
    • [constraint 제약조건명] primary key
  • FOREIGN KEY : 부모 테이블의 PK를 통해 자식 관계로 만들어줌
    • [constraint 제약조건명] references 부모테이블(부모테이블의 PK컬럼)
  • CHECK : 해당 컬럼에 잘못된 데이터가 입력 되지 않기 위한 제약이다
    • ``[constraint 제약조건명] check (제약조건 with 컬럼명)`
      • check (age >= 18)
  • DEFAULT : 제약조건은 아니지만 해당 열에 아무 값도 안넣으면 채워진다.

NOT NULL & Unique

Primary Key

2.3 Drop - DDL

테이블(및 다른 객체들 + 제약조건) 삭제

  • DROP TABLE 테이블명 : 테이블이 삭제 된후 휴지통에 들어감
  • DROP TABLE 테이블명 PURGE : 테이블 삭제시 휴지통에 안놓고 완전삭제
  • FLASHBACK TABLE 테이블명 TO BEFORE DROP : 휴지통에 들어간 테이블 복원시
  • SHOW RECYCLEBIN : 휴지통 내용보기
  • PURGE RECYCLEBIN : 휴지통 비우기
  • PURGE TABLE 테이블명 : 휴지통에 들어간 특정 테이블 완전삭제
  • DROP TABLE 부모테이블 CASCADE CONSTRAINT : 자식이 참조하더라도 강제 삭제하는 경우 // 자식테이블은 삭제 안됨

5 테이블 외에 자주 사용되는 객체

5.1 데이터 사전

테이블은 user table과 data dictionary로 나뉜다.

데이터 사전은 데이터베이스를 운영하는 데 필요한 모든 정보를 저장하는 특수 테이블이다.

5.1.1 USER_ 접두어를 가진 데이터 사전

현재 접속한 사용자가 소유한 객체에 대한 정보들

  • user_tables
    • table_name
  • user_cons_columns
  • user_indexes
    • index_name, table_name
    • index_type, table_owner, table_type, uniqueness, compression
  • user_ind_columns
    • index_name, table_name
    • column_name, column_position, column_length, char_length, descend
  • user_views
    • view_name, text
  • user_sequences
    • min_value, max_value, increment_by
  • …

5.1.2 ALL_ 접두어를 가진 데이터 사전

현재 접속한 사용자가 사용할 수 있는 모든 객체에 대한 정보들

  • all_tables
    • owner
    • table_name
  • …

5.1.3 DBA_ 접두어를 가진 데이터 사전

관리 권한을 가진 사용자만 조회할 수 있다.

  • DBA_TABLES
  • DBA_USERS

6. 오라클 주요 내장함수

6.1 문자열 관련

  • NVL(문자열, NULL인 경우 대체할 값) : 테이블 열 수준의 벡터 데이터만 처리함

  • LOWER(문자열), UPPER(문자열)

  • INITCAP(문자열) : 첫 문자를 대문자로, 나머지는 소문자로

  • CONCAT(문자열, 문자열)

  • LENGTH(문자열), LENGTHB(문자열) : 문자열 바이트 수

  • RPAD(문자열, 전체 자리수,'지정문자'), LPAD(문자열, 전체 자리수,'지정문자') : 우(좌)측을 지정한 값으로 채운다

  • INSTR(문자열, 찾을 문자열) : 문자열 인덱스 반환, 없으면 0 출력

  • SUBSTR(문자열, 시작인덱스, 길이)

  • REPLACE(문자열, 바꿀문자열, 바뀔문자열)

  • TO_NUMBER(숫자인문자열)

  • TO_CHAR

    • TO_CHAR(숫자 혹은 날짜)

    • TO_CHAR(숫자 혹은 날짜, 숫자포맷 혹은 날짜포맷)

      • 9는 값이 있으면 표시 ,없으면 표시 안함

      • 0는 값이 있으면 표시, 없으면 0으로 표시

      • 단,소수점은 9든 0이든 값이 없으면 모두 0으로 표시됨

      • 단, 실제값의 정수 자리수가 더 많으면 값이 #으로 표시됨

      • 날짜형 문법

        • yy:년도 2자리만

        • yyyy:년도 4자리

        • mm:01~12형태의 월

        • d: 요일 반환(일요일은 1,월요일 2)

        • dd:1~31형태의 일 표시

        • ddd:해당 년도의 1월 1일부터 현재까지의 일수

        • AM(혹은 PM) : 오전, 오후

        • HH : 1~12시 형태로 표시

        • HH12 : 1~12시 형태로 표시

        • HH24 : 0~23시

        • MI:0~59분

        • SS:0~59초

  • TO_DATE(문자열)

    • TO_DATE(‘2000[특수문자]10[특수문자]20’)
