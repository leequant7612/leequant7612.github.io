Posted on 2022-05-17 In Web Programming , Java

1. 톰캣 설정

Download Tomcat → 환경설정(PC) → 오라클포트변경 → CMD start.bat 실행 → 이클립스설정

1.1 Download Tomcat

  1. https://www.apache.org/
  2. Projects List에서 Tomcat선택
  3. Tomcat 9에서 64-bit Windows zip 설치
  4. C:\Program Files\Java에 압축 해제

1.2 환경설정(PC)

환경변수 설정에서 System variables에 다음을 추가한다.

1.3 오라클XE 포트 변경

톰캣서버에서 HTTP 방식의 request를 받는 프로세스(Connector)는 8080포트를 차지한다.

Oracle의 커넥터도 8080포트를 사용하므로 포트를 변경해주자.

  1. 시스템 계정으로 접속한 후에
  2. exec dbms_xdb.sethttpport(8181);
  3. select dbms_xdb.gethttpport() from dual;

1.4 톰캣 서버 실행

  1. ${CATALINA_HOME} 폴더 속성
  2. 보안탭에서 사용자에게 모든 권한 부여
  3. ${CATALINA_HOME}\bin\startup.bat 실행

2. 이클립스 설정

2.1 Dynamic Web Project 디폴트로 생성

  • Generate web.xml deployment descriptor는 체크

2.2 Server 생성

  • Server type : Tomcat 9.0
  • Server host name : localhost (개인선택)
  • Server name : JSPServer (개인선택)
  • Tomcat installation directory : ${CATALINA_HOME}
  • JRE : ${JAVA_HOME}
  • Resource : JSPProj (기존에 생성한 Dynamic Web Project 명)

서버 생성 후 Targeted Runtimes에 서버를 등록

2.3 외부 라이브러리 등록

${프로젝트}/src/WEB-INF/lib에 jar 파일을 추가

  • ojdbc6.jar : DB API
  • jstl-1.2.jar : EL과 연동되는 태그 라이브러리
  • cos.jar : MultipartRequest API를 활용한 업로드 라이브러리
    • enctype="multipart/form-data" 타입의 POST request 분석
    • 파일을 찾아내고 저장하는 역할을 한다.

2.4 Context 설정

${프로젝트}/src/WEB-INF/web.xml에서 다음의 설정을 할 수 있다.

  • 컨텍스트 초기화 파라미터
  • 세션객체의 유효 시간
  • 서블릿 등록
  • 인덱스 jsp 설정

Java Class

Posted on 2022-04-13 In Programming Language , Java

1. 제어자

클래스, 필드, 메서드, 생성자 등에게 특징을 부여하는 문법

1.1 Access modifier

  • private < package(생략가능) < protected < public
    • private: 자기 클래스내에서만 접근 가능
    • package : 같은 패키지 안에서만 접근 가능
    • protected : 다른 패키지일지라도 상속관계가 있으면 접근 가능
    • public : 어디서나 접근 가능
  • 대상 = {필드, 메서드, 생성자} + 클래스
    • 클래스의 경우 package와 public을 구분해서 쓴다.
    • 다른 패키지에서 임포트를 허용할 클래스라면, public을 써야만한다.
